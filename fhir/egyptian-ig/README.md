# HealthFlow Egyptian FHIR Implementation Guide

A FHIR R4 Implementation Guide (IG) defining the profiles, value sets, and
code systems used by the HealthFlow HCX (Health Claims Exchange) for Egypt.

This IG is **structural / starter** in nature. Clinical and payer SMEs are
expected to refine cardinalities, bindings, and terminology over time.

## Layout

```
fhir/egyptian-ig/
  sushi-config.yaml           # SUSHI configuration
  ig.ini                      # IG Publisher entry point
  build.sh                    # End-to-end build (sushi + IG Publisher)
  input/
    fsh/
      profiles/               # Resource profiles (HfcxPatient, HfcxClaim, ...)
      valuesets/              # Value sets (EgyptianGovernorateVS, ...)
      codesystems/            # Code systems (EgyptianGovernorateCS, ...)
  fsh-generated/              # Generated by sushi (gitignored)
  output/                     # Generated by IG Publisher (gitignored)
  input-cache/                # IG Publisher cache (gitignored)
```

## Prerequisites

- **Node.js** 18+ (for `npm`)
- **SUSHI**: `npm install -g fsh-sushi`
- **Java** 11+ (for the HL7 IG Publisher)
- **curl** (used by `build.sh` to download the publisher jar)

## Building

From this directory:

```bash
bash build.sh
```

This will:
1. Run `sushi .` to compile FSH into FHIR JSON resources under `fsh-generated/`.
2. Download `publisher.jar` (HL7 IG Publisher) on first run into `input-cache/`.
3. Run the IG Publisher against `ig.ini`, producing the rendered IG and a
   NPM-style FHIR package at `output/package.tgz`.

The `output/package.tgz` artifact is the canonical IG package consumed at
runtime by `hcx-apis` (and other services) for profile validation. CI uploads
it as a build artifact (see `.github/workflows/fhir-ig-build.yml`).

## Local override

To use a pre-downloaded publisher jar:

```bash
IG_PUBLISHER_JAR=/path/to/publisher.jar bash build.sh
```
