# HealthFlow Egyptian FHIR Implementation Guide

A FHIR R4 Implementation Guide (IG) defining the profiles, value sets, and
code systems used by the HealthFlow HCX (Health Claims Exchange) for Egypt.

This IG is **structural / starter** in nature. Clinical and payer SMEs are
expected to refine cardinalities, bindings, and terminology over time.

## Layout

```
fhir/egyptian-ig/
  sushi-config.yaml           # SUSHI configuration
  ig.ini                      # IG Publisher entry point (only used by --html)
  build.sh                    # End-to-end build (sushi + NPM tgz; optional HTML)
  input/
    fsh/
      profiles/               # Resource profiles (HfcxPatient, HfcxClaim, ...)
      valuesets/              # Value sets (EgyptianGovernorateVS, ...)
      codesystems/            # Code systems (EgyptianGovernorateCS, ...)
  fsh-generated/              # Generated by sushi (gitignored)
  output/                     # Generated by build.sh (gitignored)
    package.tgz               # NPM-shaped FHIR package — runtime artifact
    package/                  # Unpacked package contents
    site/                     # Browsable HTML (only with --html)
  input-cache/                # IG Publisher jar cache (gitignored, --html only)
```

## Prerequisites

- **Node.js** 18+ (for `npm`)
- **SUSHI**: `npm install -g fsh-sushi`
- **Java** 11+ — only required for the optional `--html` mode (HL7 IG Publisher)
- **curl** — only required for the optional `--html` mode

## Building

Default mode — produces the runtime NPM package only (fast, ~5 seconds):

```bash
bash build.sh
```

This will:
1. Run `sushi .` to compile FSH into FHIR JSON resources under `fsh-generated/`.
2. Generate `output/package/package.json` from `sushi-config.yaml`.
3. Copy the SUSHI-generated resources into `output/package/`.
4. Produce `output/package.tgz` in the conventional NPM FHIR layout
   (top-level `package/` directory; consumed by HAPI-FHIR's
   `NpmPackageValidationSupport`).

`output/package.tgz` is the canonical IG package consumed at runtime by
`hcx-apis` (and other services) for profile validation. CI uploads it as a
build artifact (see `.github/workflows/fhir-ig-build.yml`).

### Optional: rendered HTML site

To additionally render a browsable IG site (downloads the ~100 MB HL7 IG
Publisher jar on first run):

```bash
bash build.sh --html
```

This generates a default `input/includes/menu.xml` and `input/pagecontent/index.md`
if they don't exist, downloads the publisher jar to `input-cache/publisher.jar`,
and runs it against `ig.ini`. Useful for local previewing; not used in CI.

To use a pre-downloaded publisher jar:

```bash
IG_PUBLISHER_JAR=/path/to/publisher.jar bash build.sh --html
```
