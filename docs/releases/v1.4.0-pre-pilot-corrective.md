# v1.4.0 — Pre-Pilot Corrective Sprint

**Release date:** May 6, 2026
**Cycle:** Pre-pilot corrective follow-up to v1.3 (`docs/remediation/agentic-plan-v1.3.md`)
**Plan:** `docs/remediation/agentic-plan-v1.4.md`
**Verdict:** **PILOT-READY** (unconditional)

This cycle closed four findings surfaced by an external second-opinion
review (the Manus cross-validation pass) that ran against the post-v1.3
codebase. Two were P0 security items the prior reviews missed; one was
a P1 config mismatch with operational consequences; one was a P0
strategic gap (no SDKs exist) that converts a multi-sprint commitment
into a tracked plan.

## Gaps closed

| Gap | PR | Summary |
|---|---|---|
| **V1** — Rotate leaked Swasth-fork RSA private key | #21 | A 2048-bit PKCS#8 RSA private key was the default value of `${hcx_privateKey}` in `hcx-scheduler-jobs/.../application.yml` line 3, inherited from the upstream Swasth public repo. Default replaced with Spring's `${hcx_privateKey:?...}` required-property syntax (fail-fast at boot). Test fixture in `EventGeneratorTest` regenerated to use an inline `KeyPairGenerator` (the same compromised key was embedded as a test value). New `docs/security/known-leaked-credentials.md` documents L-001 with status, origin, used-for, reachability, rotation evidence, and "why no git-history rewrite" rationale. New CI step `Block embedded private keys` in `maven.yml` prevents recurrence. |
| **V2** — `NotificationService.java` SQL injection | #22 | Four `String.format` SQL sites (insertRecords, fetchSubscriptions, getSubscriptionById, updateSubscriptionById) were missed by the v1.2 Gap-5 sweep. Worse: `fetchSubscriptions` appended user-controlled filter values inside single quotes, allowing `topic_code = "' OR '1'='1"` to bypass the entire WHERE clause. All four sites converted to `PreparedStatement` parameter binding; filter-append loop now uses `?` placeholders. New `NotificationServiceSqlInjectionIT` (3/3 pass) captures the SQL+params via Mockito `doAnswer` and asserts the malicious value is bound, never inlined. SonarCloud S2077 enforcement comment extended to cover `hcx-apis`. |
| **V3** — Keycloak realm name mismatch | #23 | The actual realm is `hcx-egypt` (per `deployment/keycloak/hcx-egypt-realm.json`); five artifacts incorrectly used `healthflow-hcx-egypt`. Fixed in `phase3/security/application-vault.yml`, 8 occurrences in the Postman collection, `phase1/infrastructure/kubernetes-dev.yml`, `phase1/cicd/github-actions-pipeline.yml`, and `phase1/scripts/contextualize-codebase.sh`. New `RealmAssertion` `@Component` (5/5 unit tests) compares `keycloak.realm` to the realm segment of `jwt.jwkUrl` at `ApplicationReadyEvent` and refuses to start under strict mode. `instanceName` references (a different concept — the platform's HCX participant slug, not the realm) were intentionally left at `healthflow-hcx-egypt`. |
| **V4** — SDK delivery plan | #24 | Documentation-only PR. New `docs/strategy/sdk-delivery-plan.md` converts the previously-deferred SDK promise into an owned, dated commitment for Java (~6 weeks), Python (~10 weeks), .NET + JavaScript (post-GA within 90 days). GA gate: Java + Python at 1.0.0. Pilot strategy works without all SDKs by shipping pilot partners a Java SDK pre-release. `CONTRIBUTING.md` gains an "SDK delivery" subsection. Tracking issues — repo Issues are disabled; the PR description carries pre-drafted bodies for the four SDKs and the engineering lead picks Issues vs an external tracker. |

## Security posture

This cycle is unusually security-heavy: V1 + V2 are both P0. The combination closes the two highest-impact preventable issues that survived the v1.2/v1.3 cycles:

- **V1** removes a credential that any reader of the repo could trivially extract and use to forge notification events.
- **V2** removes an authenticated-network-participant injection vector that bypassed allow-list filtering entirely.

The shared lesson: the v1.2 Gap-5 SQL sweep didn't cover `hcx-apis/`, and the v1.2/v1.3 plans didn't audit production yamls for embedded keys. The v1.4 plan added two CI guardrails that prevent recurrence regardless of which contributor is reading what plan: `Block embedded private keys` (V1) and the extended S2077 scope (V2).

## What v1.4 did NOT close (tracked for v1.5)

- `ParticipantControllerTests` Indian-fixture vs `EgyptianFieldValidator.validate()` failures in `ParticipantService.java:261`. 9 tests fail; v1.3 N4 cleaned the obvious test fixtures but the deeper coupling between the participant-create flow's validation and the test bodies remains.
- `NotificationControllerTest.testSubscriptionListException` wire-format mismatch: assertion compares against `ErrorCodes.INTERNAL_SERVER_ERROR.name()` but JSON now serializes via `@JsonValue` to `"ERR-T-002"`. Pre-existing v1.2 Gap 8 leak.
- `api-gateway` Mockito-on-final-class errors. Pre-existing; needs `mockito-inline`.
- `hcx-pipeline-docker-build.yml` is `workflow_dispatch`-only and didn't auto-run on the v1.3 N3 PR. Switching to `pull_request` trigger with a `paths:` filter is a v1.5 follow-up.
- Structural split of `ErrorCodes` into three sub-enums. Wire format is already correct.
- The hardcoded `ap-south-1` JWK URL fallback in the gateway's vault profile. Tracked separately as a deployment item.
